# Un-Jira — Anti-Paralysis Task Manager

## Executive Summary

Un-Jira is a minimalist task helper designed to remove cognitive friction for people with ADHD. The app accepts a single, abstract blocking task, uses a strict LLM prompt to decompose it into a short list of atomic steps, and then forces a single-item focus interface so users can act without planning overhead.

## Core Architecture & Workflow

- Frontend: Vite + React (or Vue) PWA — single input capture, absolute-focus view.
- Backend: Node.js + Express proxy that sanitizes and forwards prompts to an LLM.
- LLM: Hosted model (e.g., Gemini 2.5 Flash) called by the backend proxy.

Flow diagram (conceptual):

[ Frontend: React/Vue PWA ] ──(LocalStorage state)
             │
             ▼  POST /api/un-jira/v1/breakdown { task }
[ Backend: Node.js / Express Proxy ]
             │
             ▼  (SDK call with strict prompt)
[ LLM API ]

Core application loop:
- State A — Capture/Inbox: single textarea input, no lists visible.
- State B — Processing: backend requests steps from LLM and sanitizes result.
- State C — Absolute Focus: UI shows only current step; completing it advances index and persists locally.

## Technical Specification

### 1) Prompt Engineering (the engine)

The backend must send a strict, no-fluff prompt to the LLM. Example prompt (Spanish):

```
Actúa como un desglosador de tareas optimizado para TDAH severo.
Tu objetivo es coger una tarea abstracta y romperla en pasos tan ridículamente pequeños y específicos que requieran cero esfuerzo mental para empezar.

Reglas estrictas:
1. Máximo 5 pasos.
2. Cada paso debe ser una acción física o cognitiva ejecutable en menos de 5 minutos.
3. No uses lenguaje motivacional ni consejos redundantes. Va directo a la acción.
4. El primer paso debe ser absurdamente fácil (ej: "Abre el editor de código", "Levántate y coge una bayeta").

Formato de salida: Devuelve EXCLUSIVAMENTE un array JSON plano de strings, sin Markdown, sin bloques de código, solo el JSON.
```

Example expected LLM output (exact JSON array):

```json
[
  "Recoge las tazas sucias y déjalas en la cocina.",
  "Tira los papeles acumulados a la basura.",
  "Pasa un trapo húmedo por la superficie.",
  "Coloca el teclado y el ratón en su sitio."
]
```

### 2) Backend Endpoint — POST /api/un-jira/v1/breakdown

- Technology: Node.js + Express (or Python + FastAPI).
- Payload input: `{ "task": "string" }` (JSON)
- Payload output: `{ "steps": ["string", ...] }` (JSON)
- Behavior: call LLM with the strict prompt, then sanitize the model output to ensure a raw JSON array (strip code fences, markdown, etc.). Return `steps` as an array of strings.

Sanitization notes:
- Remove surrounding ```json or other fences.
- If the model returns a single string, try to parse JSON from it; if parsing fails, attempt a tolerant extraction (best-effort). If still failing, return a 502 with a helpful error.

### 3) Frontend & PWA layer

- Stack: Vite + React (or Vue) + Tailwind CSS.
- PWA plugin: `@vitejs/plugin-react` + `@vite-pwa/plugin` configured for `display: standalone`.
- Local state keys: `unjira_steps` (JSON array) and `unjira_index` (integer) in `localStorage`.
- Views: `CaptureView` (single textarea + submit) and `FocusView` (single-line step display + complete button).
